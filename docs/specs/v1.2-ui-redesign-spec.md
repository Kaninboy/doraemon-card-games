# v1.2 UI Redesign Spec — "เกมไพ่วงเหล้า" Drinking Party Edition

## Restriction
- Must not change my game title "เกมไพ่โดเรม่อน" and "Doraemon Card Games"

## Design Vision

**Aesthetic direction:** Monochrome party — a bold black-and-white foundation punctuated with warm amber/gold accents (the color of beer). The feel is a late-night drinking game: slightly irreverent, tactile, fun. Not corporate, not childish — somewhere between a speakeasy menu and a party invitation.

**Memorable detail:** Every card draw feels like a mini-event. The 3D card flip from a custom party-themed card back makes each draw theatrical. Subtle beer/party iconography scattered throughout keeps the drinking game identity front and center without being garish.

**Typography direction:**
- **EN display (titles, headings):** **Lilita One** — bold, chunky, high-impact poster feel. Used for all EN-mode titles and headings.
- **TH display (titles, headings):** **Mitr SemiBold/Bold** — since Lilita One has no Thai support, Mitr handles all Thai-mode titles and headings. Its rounded, compact style pairs well with Lilita One's weight.
- **Body (both languages):** **Mitr Regular/Medium** — card rules, subtitles, buttons, modal text in both EN and TH.
- Font strategy: `font-family: 'Lilita One', 'Mitr', sans-serif` for display elements — Lilita One handles Latin glyphs, Mitr falls back for Thai characters automatically.

**Color palette:**
```
Primary:        #0a0a0a (near-black)
Primary Light:  #ffffff (white)
Accent Warm:    #F5A623 (beer amber/gold)
Accent Glow:    #FFD580 (light amber, for glows/highlights)
Accent Dark:    #C4841D (dark amber, for hover states)
Danger/Red:     #E74C3C (for Kings, special events)
Surface Dark:   #1a1a1a (cards, modals in dark mode)
Surface Light:  #f5f5f0 (slight warm off-white for light mode)
```

CSS variables extended — not replaced. Amber accent added as `--accent-color`, `--accent-glow`, `--accent-dark`.

### Task 5: Start Game — Direct Navigation

**Current:** `StartPage` → click "Start Game" → `RulesModal` appears → click "Start Game" inside modal → `GamePage`.

**New:** `StartPage` → click "Start Game" → `GamePage` immediately.

**Implementation:**
- `StartPage`'s "Start Game" button calls `onStartGame()` directly (the prop that navigates to game page).
- Remove the intermediate modal flow from the start button.
- Rules are now accessible via the separate "Rules" button (Task 3).

### Task 6: Game Page — Fix Button/Rule Box Mobile Layout

**Current issue:** On mobile, the "Draw Card" button can be shorter in width than the rule text box above it, creating visual misalignment.

**Fix:**
- Game action area (rule box + buttons) should be in a container with consistent width.
- Buttons should be `width: 100%` within this container on mobile.
- Rule text box and buttons share the same max-width container (e.g., `max-width: 400px; width: 100%`).
- At tablet+ breakpoints, buttons can sit side-by-side but still fill the container width.

## Additional Visual Enhancements (Part of v1.2 Redesign)

### Enhancement A: 3D Card Flip Animation

**Concept:** When the user draws a card, the card starts face-down (showing the party-themed card back) and flips 180° on the Y-axis to reveal the card face. This is a CSS-only 3D transform.

**Implementation approach:**
- `.card-container` — sets `perspective: 1000px`
- `.card-inner` — has `transform-style: preserve-3d`, `transition: transform 0.6s ease-in-out`
- `.card-front` and `.card-back` — both use `backface-visibility: hidden`; `.card-back` is `rotateY(0)`, `.card-front` is `rotateY(180deg)` by default
- When card is "flipped" (state: `isFlipped`), `.card-inner` gets `transform: rotateY(180deg)`
- On each new draw: reset `isFlipped` to `false` (shows back), then after a short delay (~100ms) set `isFlipped` to `true` to trigger the flip.

**Card back design (CSS-rendered):**
- Background: `#0a0a0a` (dark) or `#1a1a1a`
- Border: 2px solid `var(--accent-color)` (amber)
- Center icon: Beer mug emoji or SVG, rendered in amber
- Corner decorations: Small suit symbols (♠♥♦♣) in amber, subtle
- Pattern: Optional subtle repeating dot or cross-hatch pattern via CSS `background-image`

### Enhancement B: Party Theme Elements

**Beer mug accents:**
- Title on Start Page flanked by beer mug emojis: `🍺 เกมไพ่โดเรม่อน 🍺`
- Or use a CSS-rendered beer mug icon if emoji rendering is inconsistent across platforms.
- Consider: small beer mug icon next to the "cards remaining" counter (as if cards are "drinks remaining").

**Subtle background texture:**
- Both themes get a very subtle noise/grain texture overlay via CSS (`background-image` with a tiny noise PNG or SVG pattern at very low opacity — 0.03–0.05).
- This adds a tactile, printed-material quality that elevates beyond flat digital.

**Button styling — party edition:**
- Primary buttons: Amber background (`--accent-color`), dark text, slightly rounded (border-radius: 12px).
- Hover: Glow effect — `box-shadow: 0 0 20px rgba(245, 166, 35, 0.3)`.
- Active: `translateY(1px)` press-down effect.
- Secondary buttons: Transparent with amber border, amber text.

**Card display polish:**
- Slightly larger card on desktop (cap at 240px width instead of 200px).
- Enhanced shadow: multi-layered box-shadow for depth.
- Suit symbols in proper red/black with slightly richer colors.
- King cards get a special amber glow border to signal "this is a big deal."

### Enhancement C: Game State Transitions

**Between states (initial → playing → game over):**
- Fade-in transition for new content (`opacity 0→1, translateY(10px→0)` over 300ms).
- CSS `@keyframes fadeInUp` applied to game content on state change.

**Game Over state:**
- Larger, bolder "Game Over" text with a subtle amber glow.
- "Play Again" button is prominent (amber, large).
- Optional: confetti-style CSS animation (a few small squares/circles floating down via `@keyframes`).

### Enhancement D: Responsive Polish

**Mobile (≤480px):**
- Card stack on Start Page scales down proportionally.
- Buttons stack vertically, full-width.
- Controls (theme/language) remain top-right but with adequate spacing.
- Minimum tap target: 44px maintained everywhere.

**Tablet (481px–768px):**
- Card stack slightly larger.
- Buttons can sit side-by-side.

**Desktop (769px+):**
- Max content width: 600px centered.
- Card stack at full size with hover animation.
- More generous spacing.
