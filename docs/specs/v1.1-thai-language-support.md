# v1.1 — Thai Language Support

## Overview

Add an English/Thai language toggle to the Doraemon Card Game. Players can switch the UI and all card rules between English (EN) and Thai (TH) at any time. The selected language persists across sessions via localStorage.

**Version:** 1.1  
**Status:** Spec  
**Affected pages:** Start Page, Game Page, Rules Modal

---

## Goals

- All visible UI strings are translatable (titles, buttons, instructions, card rules, king rules)
- Language preference persists in localStorage
- Toggle is always accessible alongside the theme toggle
- No new runtime dependencies

## Non-Goals

- Translating error messages or dev-only text
- Supporting languages beyond EN and TH
- Server-side i18n or URL-based locale routing

---

## Architecture

Mirrors the existing ThemeContext pattern exactly — three files per context.

### New Files

| File | Purpose |
|------|---------|
| `src/i18n/translations.ts` | All EN + TH strings (UI + card rules + king rules) |
| `src/context/LanguageContextDef.ts` | `Language` type + `LanguageContextType` interface + `createContext` |
| `src/context/LanguageContext.tsx` | `LanguageProvider` — reads/writes localStorage, exposes `language` + `toggleLanguage` |
| `src/context/useLanguage.ts` | Hook to consume `LanguageContext` |
| `src/components/LanguageToggle.tsx` | EN/TH toggle button component |

### Modified Files

| File | Change |
|------|--------|
| `src/App.tsx` | Wrap app with `<LanguageProvider>` alongside `<ThemeProvider>` |
| `src/components/StartPage.tsx` | Replace hardcoded strings with `t.*` lookups |
| `src/components/GamePage.tsx` | Replace hardcoded strings with `t.*` lookups |
| `src/components/RulesModal.tsx` | Replace hardcoded strings with `t.*` lookups |
| `src/types/index.ts` | Remove `cardRules` and `kingRules` exports (moved to `translations.ts`) |
| `src/utils/gameState.ts` | Remove `getCardRule` or reduce to rank-key lookup only |

### localStorage Key

```
doraemon-card-game-language   →  'en' | 'th'
```

Default: `'en'`

## Context Implementation

### `src/context/LanguageContextDef.ts`

```ts
import { createContext } from 'react';
import type { Language } from '../i18n/translations';

export type { Language };

export interface LanguageContextType {
  language: Language;
  toggleLanguage: () => void;
}

export const LanguageContext = createContext<LanguageContextType | undefined>(undefined);
```

### `src/context/LanguageContext.tsx`

```tsx
import { useState, type ReactNode } from 'react';
import { LanguageContext, type Language } from './LanguageContextDef';

const LANGUAGE_STORAGE_KEY = 'doraemon-card-game-language';

export function LanguageProvider({ children }: { children: ReactNode }) {
  const [language, setLanguage] = useState<Language>(() => {
    const saved = localStorage.getItem(LANGUAGE_STORAGE_KEY);
    return (saved as Language) || 'en';
  });

  const toggleLanguage = () => {
    const next: Language = language === 'en' ? 'th' : 'en';
    localStorage.setItem(LANGUAGE_STORAGE_KEY, next);
    setLanguage(next);
  };

  return (
    <LanguageContext.Provider value={{ language, toggleLanguage }}>
      {children}
    </LanguageContext.Provider>
  );
}
```

### `src/context/useLanguage.ts`

```ts
import { useContext } from 'react';
import { LanguageContext } from './LanguageContextDef';

export function useLanguage() {
  const context = useContext(LanguageContext);
  if (context === undefined) {
    throw new Error('useLanguage must be used within a LanguageProvider');
  }
  return context;
}
```

## Using Translations in Components

Each component accesses translated strings through `t`, which is the current locale's sub-object:

```ts
import { useLanguage } from '../context/useLanguage';
import { translations } from '../i18n/translations';

// Inside component:
const { language } = useLanguage();
const t = translations[language];

// Usage examples:
<button>{t.drawCard}</button>
<p>{t.cardRules['A']}</p>
<p>{t.kingRules[kingsDrawn - 1]}</p>
```

No wrapper function needed — `t` is the locale sub-object directly.

---

## `getCardRule` in `src/utils/gameState.ts`

`getCardRule` currently reads from `cardRules`/`kingRules` in `src/types/index.ts`. After this change those constants are removed.

**Preferred approach:** Delete `getCardRule`. Components read rules directly via `t.cardRules[card.rank]` and `t.kingRules[kingsDrawn - 1]`. This keeps language-awareness in the view layer, not in game logic.

## Language Toggle Component

`src/components/LanguageToggle.tsx` — renders a button showing the *inactive* language label (`EN` when Thai is active, `TH` when English is active) and calls `toggleLanguage()` on click. Placed alongside `ThemeToggle` in the corner controls area.

## Acceptance Criteria

- [ ] Language toggle visible on both Start Page and Game Page
- [ ] Clicking the toggle switches all UI strings immediately (no reload)
- [ ] Card rules and king rules display in the selected language
- [ ] RulesModal shows rules in the selected language
- [ ] Language preference survives page refresh (key: `doraemon-card-game-language`)
- [ ] Default language is English
- [ ] Dark/light theme works independently of language
- [ ] Game state (deck, drawn cards) is unaffected by language switch

- Per-card rule overrides outside the translations object
